# What is a CLASS really?

A class is a **blueprint/template** for a type of thing.

Blackjack example:

* Card
* Deck
* Player
* Dealer
* Hand
* Game

Those are classes because they represent *real entities* in the system.

Example:

```java
class Card {
    String suit;
    String rank;
}
```

This says:

> “A card has a suit and a rank.”

---

# What is an OBJECT?

An object is an actual instance of a class.

```java
Card c1 = new Card();
```

Now `c1` is a REAL card object.

Think:

* Class = recipe
* Object = actual cake

or

* Class = blueprint
* Object = actual house

# Interfaces

This confuses everyone at first.

An interface is basically:

> “Any class that promises to do these actions.”

Like a contract.

Example:

```java
interface Drawable {
    void draw();
}
```

Now ANYTHING can implement it:

```java
class Card implements Drawable
class Button implements Drawable
class Enemy implements Drawable
```

Each has its own version of `draw()`.

---

# Real Mental Model for Interfaces

Now:

can ALL be swapped interchangeably.

That’s professional OOP.

# Streams

Streams are NOT OOP specifically.

They’re more like:

> “Data pipelines for collections.”

Without streams:

```java
for (Card c : cards) {
    if (c.getValue() > 10) {
        ...
    }
}
```

With streams:

```java
cards.stream()
     .filter(c -> c.getValue() > 10)
     .forEach(System.out::println);
```

Streams let you describe:

* filtering
* mapping
* transforming
* aggregating

in a clean pipeline.
